# Security Policy

HAL Recovery Compiler is a simulation-only proof of concept.

- Do not use it to control real hardware.
- Do not use it as a chip, safety, or production certification tool.
- Do not load generated profiles into firmware, drivers, or memory controllers.
- Avoid sharing proprietary chip maps, ATE/BIST output, credentials, or sensitive artifact data in public issues.

Report security concerns through GitHub issues or the maintainer contact if one is later published in repository metadata. Redact sensitive data and provide the minimum synthetic reproduction needed.

Every generated report remains candidate-only, human-review-required, hardware-control-disabled, and `simulation_only_not_certified`.
